📋 1. Introduzione

Il modulo Navision di FTC HUB permette di aprire sessioni Remote Desktop (RDP) verso Microsoft Dynamics NAV 2009 R2 direttamente dal browser, senza dover inserire manualmente le credenziali ad ogni accesso.

Le credenziali sono gestite centralmente dall'IT e assegnate per department. Ogni utente vede solo le utenze di competenza del proprio ufficio.

Accesso al modulo: il modulo Navision è visibile nella sidebar solo agli utenti per cui è stato abilitato da Gestione Utenti → Accesso Moduli.
DepartmentCosa vede
ITTutte le utenze di tutti i department — può scegliere quale usare
FINANCE, COMMERCIAL, DM…Solo le proprie utenze di department
⚙️ 2. NAV Agent — installazione

Per poter aprire le sessioni RDP dal browser è necessario installare il NAV Agent: un piccolo programma che gira in background sul proprio PC e fa da tramite tra il browser e il client RDP di Windows.

Operazione da fare una sola volta per PC. Una volta installato, l'agent si avvia automaticamente ad ogni accensione del computer.
  1. Dalla pagina Navision di FTC HUB, clicca il pulsante Scarica Agent (compare solo se l'agent non è attivo).
  2. Il browser (Edge) mostrerà un avviso di sicurezza sul file scaricato. Clicca i tre puntini (…) accanto al file → Mantieni → Mostra altri → Mantieni comunque.
    Questo avviso è normale per qualsiasi programma non firmato con certificato commerciale. Il file è sicuro — è distribuito direttamente da FTC HUB.
  3. Apri il file ftchub-nav-agent.exe scaricato. Si aprirà una finestra nera (terminale) che conferma che l'agent è in ascolto. Puoi minimizzarla — non chiuderla.
  4. Torna sulla pagina Navision in FTC HUB e clicca l'icona di aggiornamento (🔄) accanto allo stato. Il badge diventerà Agent OK.
Avvio automatico: dalla prossima accensione del PC, l'agent si avvierà da solo — non sarà necessario riaprirlo. La voce viene aggiunta automaticamente alla chiave di avvio di Windows (HKCU\...\Run) al primo lancio.
🚀 3. Primo avvio dopo riavvio PC

Dopo il riavvio del computer, l'agent parte in background automaticamente. Non è necessaria nessuna azione.

Se la pagina Navision mostra Agent offline dopo un riavvio, significa che l'agent non si è avviato. In questo caso:

  • Cerca ftchub-nav-agent.exe nella cartella Download e riaprilo
  • Oppure contatta l'IT per verificare l'installazione
🖥️ 4. Aprire una sessione NAV

Dalla pagina Navision sono disponibili i pulsanti per i 5 ambienti:

PulsanteAmbiente
NAV IT01 — CLASSICNavision IT01, interfaccia classica
NAV IT02 — CLASSICNavision IT02, interfaccia classica
NAV IT02 — NEWNavision IT02, nuova interfaccia
NAV IT03 — CLASSICNavision IT03, interfaccia classica
NAV IT03 — NEWNavision IT03, nuova interfaccia

Per aprire una sessione: clicca il pulsante dell'ambiente desiderato. La sessione RDP si apre senza chiedere username e password.

Utenti IT: se per un ambiente sono configurate più utenze (es. utenza Finance, utenza Commercial, utenza IT personale), apparirà una finestra di selezione per scegliere quale usare. Questo è utile quando un'utenza è già occupata da un'altra sessione attiva.

Il contatore "X sessioni attive" nella barra superiore mostra quante finestre RDP sono aperte in quel momento sul PC.

🔴 5. Chiudere le sessioni

Le sessioni RDP possono essere chiuse normalmente dalla finestra di Navision stessa (disconnetti / chiudi).

In alternativa, il pulsante Chiudi tutte nella barra superiore chiude immediatamente tutte le sessioni RDP aperte sul PC con un solo clic.

Attenzione: il pulsante "Chiudi tutte" chiude qualsiasi finestra mstsc.exe aperta sul PC, incluse eventuali connessioni RDP non legate a Navision.
🔑 6. Gestione credenziali Solo IT

Gli utenti IT possono aggiungere, modificare ed eliminare le configurazioni di accesso NAV direttamente dalla pagina Navision.

Aggiungere una configurazione:

  1. Clicca + Aggiungi in alto a destra nella sezione "Apri sessione NAV".
  2. Seleziona il Department di riferimento e l'Ambiente NAV (es. IT02_CLASSIC).
  3. Inserisci il Server RDP, l'Username e la Password.
  4. Facoltativamente aggiungi un'etichetta descrittiva (es. "NAV IT02 — Finance HQ").
  5. Clicca Crea configurazione.

Modificare o eliminare: sotto ogni pulsante ambiente compare il link gestisci. Cliccalo per espandere la lista delle credenziali configurate, con le icone matita (modifica) e cestino (elimina).

Le password sono salvate nel database in forma cifrata (Fernet AES-128). Nessun utente — nemmeno IT — può leggere la password in chiaro dall'interfaccia: può solo sostituirla.

Criteri per l'assegnazione delle utenze:

IT
Vede tutte le utenze di tutti i department
FINANCE
Vede solo utenze department FINANCE
COMMERCIAL
Vede solo utenze department COMMERCIAL
DM
Vede solo utenze department DM
Altri HO
Vede solo utenze del proprio department
❓ 7. Domande frequenti

Il pulsante mostra "nessuna config" — cosa significa?
Non è stata ancora configurata nessuna utenza per quell'ambiente e il tuo department. Contatta l'IT.

La sessione RDP si apre ma chiede comunque le credenziali.
Può succedere se il server RDP richiede autenticazione a due fattori o se le credenziali salvate sono scadute. Contatta l'IT per aggiornare la password.

L'agent mostra "offline" anche dopo averlo aperto.
Controlla che la finestra nera dell'agent sia ancora aperta (non chiusa). Se è aperta ma il badge rimane offline, verifica che nessun firewall o antivirus stia bloccando la porta 9999 in locale.

Come disinstallo l'agent?
Chiudi la finestra dell'agent, poi rimuovi manualmente la voce dal registro di Windows: apri regedit → naviga su HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run → elimina la chiave FTCHubNavAgent.

Devo reinstallarlo dopo un aggiornamento?
Solo se l'IT comunica una nuova versione dell'agent. Scarica il nuovo ftchub-nav-agent.exe dalla pagina Navision, sostituisci il vecchio file e riaprilo una volta — si aggiornerà da solo nel registro.
